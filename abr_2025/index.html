Essa é a página da nossa Live!

Boas vindas à página da nossa live! Cada etapa criada aqui pode ser reproduzida por você, atenção aos comandos utilizados e a ordem deles.

Os primeiros comandos foram na etapa de configuração, depois de feito não há necessidade de refaze-lo, a não ser que queira mudar as configurações.
(Configuração de usuária)
git config --global user.name Digite seu nome
git config --global user.email Digite seu email
(Configuração da Branch inicial)
git config --global init.defaultBranch main

A próxima etapa será iniciar esse projeto no Git, com o comando:
git init
Veja que seu terminal alterou! Veja na pasta do projeto (arquivos ocultos) que uma pasta .git foi criada também.
Agora você pode começar a salvar as alterações do seu código.

**Sempre que algum comando não funcionar, verifique no terminal qual erro ocorreu, pode ser algum comando digitado errado ou algum erro de sintaxe no código.

Vamos utilizar os comandos para salvar a primeira versão desse código, sem fazer alterações ainda.
Pense que você está organizando documentos em caixa, a primeira etapa será colocar os documentos na caixa, então use os comandos:
(Para verificar todos os documentos alterados)
git status
(Para adicionar todos documentos na caixa)
git add .

Agora que você já adicionou os documentos na caixa, adicione uma etiqueta na caixa, para lembrar o que foi colocado dentro:
(Para atiquetar a caixa)
git commit -m "Adicionei o projeto inicial"

Você pode verificar o histórico do que foi adicionado na caixa e informações de quem etiquetou, com o comando:
git log

Agora que você já tem a primeira versão do seu projeto, vamos executar as duas tarefas vistas na apresentação.
• Alterar paleta de cores
• Adicionar formulário de inscrição
Para cada tarefa, você irá criar uma nova branch, para que o projeto principal não seja afetado.
Vamos dar o nome de "cores" e "form" para as branch.
(Para criar uma nova branch)
git branch cores
git branch form
Note que ambas são cópias da branch main

Você pode verificar todas as branches existentes e alternar entre elas:
(Para ver a lista de branches)
git branch
(Para alternar entre as branches)
git checkout cores

Veja no terminal que você está na branch cores, faça as alterações necessárias no código para essa tarefa e salve novamente, como nos comandos que aprendemos anteriormente.
Para essa tarefa, vá até o arquivo styles.css e altere as cores conforme goste.
**Lembre-se de organizar os documentos na caixa e etiquetar a caixa, para não perder o que foi feito.
(Para verificar todos os documentos alterados)
git status
(Para adicionar todos documentos na caixa)
git add .
(Para atiquetar a caixa)
git commit -m "Alterei a paleta de cores"
(Para ver o histórico)
git log

Com essa tarefa finalizada, faça a próxima tarefa, que é adicionar o formulário de inscrição.
(Para alternar de branch)
git checkout form
No arquivo index.html, adicione o código do formulário de inscrição ao final da página.
```< form>
< label for="name">Nome:
< input type="text" id="name" name="name">
< label for="email">Email:
< input type="email" id="email" name="email">
< input type="submit" value="Enviar">
< /form>```
**Lembre-se de organizar os documentos na caixa e etiquetar a caixa, para não perder o que foi feito.
(Para verificar todos os documentos alterados)
git status
(Para adicionar todos documentos na caixa)
git add .
(Para atiquetar a caixa)
git commit -m "Adicionei o formulário de inscrição"
(Para ver o histórico)
git log
** Se o histórico estiver grande, pode precisar usar o comando :q para sair do log no terminal.

Finalizadas as duas tarefas, volte para a branch main e veja o que acontece.
(Para voltar para a branch principal)
git checkout main

Veja que o código não está atualizado, pois você ainda não fez o merge das branches cores e form com a branch main.
(Para fazer o merge)
git merge cores
git merge form
(Para ver o histórico)
git log

Com o código atualizado, você pode conectar com um repositório remoto.
Vamos usar o GitHub como exemplo, mas você pode usar qualquer outro repositório remoto.
Precisamos criar um repositório remoto, para isso, entre no GitHub e crie um repositório com o nome do projeto.
Depois de criar o repositório, copie o link do repositório remoto.
(Para conectar o repositório local com o remoto)
git remote add origin link
(Para verificar se o repositório remoto foi adicionado)
git remote -v
(Para enviar o código para o repositório remoto)
git push -u origin main
** Se for a primeira vez conectando com o GitHub, pode precisar de algumas etapas de autenticação.

Pronto, nosso código está pronto para ser distribuido e outras pessoas podem contribuir com o projeto.
Faça um teste com esse repositório, faça um clone do repositório remoto para você e faça suas alterações.
Crie uma pasta de projetos, utilize o comando:
(Para clonar o repositório remoto)
git clone https://github.com/bugelseif/site-live.git
(Para entrar na pasta do projeto)
cd site-live
(Para criar uma nova branch)
git branch seu_nome
(Para alternar entre as branches)
git checkout seu_nome
**Faça as alterações que desejar no código
(Para verificar todos os documentos alterados)
git status
(Para adicionar todos documentos na caixa)
git add .
(Para atiquetar a caixa)
git commit -m "Alterei as cores"
(Para enviar o código para o repositório remoto)
git push -u origin seu_nome

Agora você pode fazer um pull request no repositório remoto, para que o dono do repositório possa aceitar suas alterações.
Entre no repositório remoto e clique em "Compare & pull request".
Depois clique em "Create pull request".

Tem outro comando importante quando temos repositórios remotos e queremos atualizar o código, puxando as alterações do repositório remoto para o repositório local.
(Para puxar as alterações do repositório remoto)
git pull origin main

Agora você conhece os comandos básicos para trabalhar com git e conectar em repositórios remotos.
Continue praticando!
